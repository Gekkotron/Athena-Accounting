# v1.0.0-desktop-rc1 — release candidate

Second desktop pre-release after `v1.0.0-desktop-beta1`. Packaging story is
unchanged (Tauri 2 + directory-based sidecar, no code-signing yet on macOS —
right-click → **Open** on first launch). Everything since beta1 is code
hardening, security fixes, and pre-public-release repo polish rather than
new packaging surface.

## macOS Gatekeeper — required `xattr -cr` step

The `.app` inside the `.dmg` is ad-hoc code-signed (no Apple Developer
ID yet). macOS 14 (Sonoma) tightened Gatekeeper and macOS 15 / 26
(Sequoia / Tahoe) tightened it further: any browser-downloaded artifact
carries a `com.apple.quarantine` extended attribute, and on those OS
versions Gatekeeper refuses to launch ad-hoc-signed quarantined apps
with the misleading *"« Athena Accounting » est endommagé et ne peut
pas être ouvert"* dialog. Right-click → Open no longer bypasses this.

The one-shot cure is a single Terminal command that strips the
quarantine attribute:

1. Open the `.dmg` and drag **Athena Accounting** into `/Applications`.
2. Open **Terminal** and run:
   ```bash
   xattr -cr "/Applications/Athena Accounting.app"
   ```
3. Double-click the app in Finder. It launches normally from now on.

Older macOS (12/13) that still accepts ad-hoc bundles will show the
softer *"unidentified developer"* dialog — in that case, right-click →
**Open** → confirm **Open** and macOS remembers the choice.

See `docs/users/desktop-install.md` for the full walkthrough.

The three checkboxes below are ticked by a human after installing each
artifact on the corresponding OS and confirming the app launches and
`/health` returns `{ok:true}`.

- [ ] macOS — `.dmg` installs, first launch bypasses Gatekeeper via
      right-click → Open, main window loads, sidecar `/health` returns ok
- [ ] Linux — `.AppImage` runs (`chmod +x` if needed), main window loads,
      sidecar `/health` returns ok

      sidecar `/health` returns ok
